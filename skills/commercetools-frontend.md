## How to Use This Skill
1. Identify which approach you are using (commercetools Frontend or headless Next.js)
2. Load the relevant architecture file first
3. Then load the page-type-specific reference for your current task
4. Reference files contain correct/incorrect code pairs, checklists, and pitfall warnings

Progressive loading -- only load what you need:
- Commercetools Frontend extensions/actions: Load [ct-frontend-extensions.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/ct-frontend-extensions.md)
- Commercetools Frontend components/Studio: Load [ct-frontend-components.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/ct-frontend-components.md)
- Headless Next.js storefront: Load [storefront-architecture.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/storefront-architecture.md)
- PLP (filtering, pagination): Load [product-listing-pages.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/product-listing-pages.md)
- PDP (variants, product cards): Load [product-detail-pages.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/product-detail-pages.md)
- Cart UI: Load [cart-ui.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/cart-ui.md)
- Checkout flows: Load [checkout-ui.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/checkout-ui.md)
- Performance/SEO: Load [frontend-performance.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/frontend-performance.md) or [frontend-seo.md](https://github.com/ariessolutionsio/composable-skills/blob/HEAD/skills/commercetools-frontend/references/frontend-seo.md)
